# 更新日志

本文档记录 AI助理优化工具 的主要版本变化。

## 2.0.11

- `config配置优化` 改回从云端下载 `config.yaml` 并覆盖本地文件的方式。
- 下载会先写入临时文件，确认文件非空后再覆盖本地配置。

## 2.0.10

- 启动服务、重启服务、AI 助理升级完成后的启动判断不再检查 `python.exe`。
- 改为等待 Edge 新窗口或新标签页打开后再显示执行成功，避免服务页面还没弹出就提前提示成功。

## 2.0.9

- `config配置优化` 新增修改 `compression.enabled: true`。
- 只修改顶层 `compression:` 区块下的 `enabled` 字段，避免误改其他同名字段。
- 生成文件改为同时输出带版本号的 `AIOptimizeTool_v2.0.9.exe` 和固定升级名 `AIOptimizeTool.exe`。

## 2.0.8

- 修复旧版本可能把 config 参数误写成 `$10.5`、`$115`、`$1198000` 后导致保存失败的问题。
- 增加对损坏 config 参数行的自动修复逻辑。

## 2.0.7

- 优化 `config配置优化` 的字段替换方式，只修改目标字段冒号后的值。
- 保留 `config.yaml` 其他内容不变。

## 2.0.6

- `config配置优化` 移除 `hermes config migrate` 命令。
- 改为直接修改本地 `config.yaml` 中的 `context_length`、`threshold`、`protect_last_n`。

## 2.0.5

- 调整 config 优化流程，即使迁移命令失败也继续尝试修改指定参数。

## 2.0.4

- 界面标题增加工具版本号。
- 工具升级前增加用户提示，避免升级时界面关闭后用户不清楚后台正在处理。

## 2.0.3

- config 优化改为先执行配置迁移，再修改本地参数。

## 2.0.2

- `全部执行` 改名为 `一键优化`，并移动到账号信息绑定区域。
- `系统升级` 改名为 `AI助理升级`。
- 优化功能按钮的排版、配色和视觉样式。

## 2.0.1

- 增加工具原地升级功能。
- 增加启动服务、停止服务、重启服务功能。
- 增加升级进度提示。
- 优化 AI 助理升级流程，支持下载压缩包并解压安装。
